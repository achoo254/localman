## 1. @tauri-apps/plugin-updater JS API

### Installation
```bash
pnpm add @tauri-apps/plugin-updater
```

### TypeScript Function Signatures

```typescript
// Main function - check for updates
check(options?: CheckOptions): Promise<Update | null>

// Update instance methods
update.download(onEvent?: (progress) => void, options?: DownloadOptions): Promise<void>
update.downloadAndInstall(onEvent?: (progress) => void, options?: DownloadOptions): Promise<void>
update.install(): Promise<void>
update.close(): Promise<void>

// Supporting interfaces
interface CheckOptions {
  headers?: Record<string, string>
  timeout?: number
  proxy?: string
  target?: string  // override target identifier
  downgrade?: boolean  // allow downgrading
}

interface DownloadOptions {
  headers?: Record<string, string>
  timeout?: number
}
```

### Basic Usage Pattern
```typescript
import { check } from '@tauri-apps/plugin-updater'
import { relaunch } from '@tauri-apps/plugin-process'

const update = await check()
if (update?.shouldUpdate) {
  await update.downloadAndInstall()
  await relaunch()  // restart app
}
```

## 2. tauri-apps/tauri-action GitHub Actions

### Inputs for Updater Setup
```yaml
- uses: tauri-apps/tauri-action@v1
  with:
    uploadUpdaterJson: true                    # generates latest.json
    uploadUpdaterSignatures: true              # uploads .sig files
    releaseId: ${{ github.event.release.id }}  # link to release
    tagName: v${{ needs.get-version.outputs.version }}  # explicit tag (recommended)
    projectPath: ./                            # app root
```

### How It Generates latest.json
- Automatically creates manifest during build if `uploadUpdaterJson: true`
- Includes all platform binaries (windows-x86_64, darwin-aarch64, darwin-x86_64, linux-x86_64)
- Reads signatures from .sig files generated during build
- Supports `{os}-{arch}-{installer}` keys for multiple formats per platform
- Requires `tauri-plugin-updater` >=2.10.0 for multi-installer support

### Key Inputs
| Input | Purpose |
|-------|---------|
| `releaseId` | Link action output to existing GitHub release |
| `tagName` | Explicit version tag (pair with releaseId for reliability) |
| `uploadUpdaterJson` | Generate & upload latest.json |
| `uploadUpdaterSignatures` | Upload .sig files |
| `uploadPlainBinary` | Upload non-signed binaries |
| `updaterJsonPreferNsis` | Prefer NSIS format for Windows |

## 3. Tauri v2 Updater Config (tauri.conf.json)

### Minimal Configuration
```json
{
  "plugins": {
    "updater": {
      "active": true,
      "pubkey": "dW50cnVzdGVkIGNvbW1lbnQ6IG1pbmlzaWduIHB1YmxpYyBrZXk6IDE5QzMxNjYwNTM5OEUwNTgKUldSWTRKaFRZQmJER1h4d1ZMYVA3dnluSjdpN2RmMldJR09hUFFlZDY0SlFqckkvRUJhZDJVZXAK",
      "endpoints": ["https://releases.example.com/{{target}}/{{current_version}}"]
    }
  }
}
```

### Full Configuration Options
```json
{
  "plugins": {
    "updater": {
      "active": true,                    // boolean, default: false
      "dialog": true,                    // show update dialog, default: true
      "pubkey": "base64-encoded-key",   // REQUIRED - generated by tauri signer
      "endpoints": [                     // REQUIRED - supports dynamic vars
        "https://releases.example.com/{{target}}/{{arch}}/{{current_version}}"
      ]
    }
  },
  "build": {
    "createUpdaterArtifacts": true    // or "v1Compatible" for legacy apps
  }
}
```

### Dynamic Variables
- `{{target}}` - platform triple (windows-x86_64, darwin-aarch64, linux-x86_64)
- `{{arch}}` - architecture (x86_64, aarch64)
- `{{current_version}}` - app version from package.json

### Tauri Build Setup (src-tauri/Cargo.toml)
```toml
[dependencies]
tauri-plugin-updater = "2"
```

### Rust Setup (src-tauri/src/main.rs)
```rust
fn main() {
  tauri::Builder::default()
    .plugin(tauri_plugin_updater::Builder::new().build())
    .run(tauri::generate_context!())
    .expect("error while running tauri application");
}
```

## 4. latest.json Manifest Format

### Structure
```json
{
  "version": "1.0.0",
  "notes": "Bug fixes and performance improvements",
  "pub_date": "2026-03-15T10:30:00Z",
  "platforms": {
    "windows-x86_64": {
      "signature": "-----BEGIN SIGNATURE-----\nbase64...content...\n-----END SIGNATURE-----",
      "url": "https://github.com/org/repo/releases/download/v1.0.0/app-x64-setup.nsis.zip"
    },
    "windows-aarch64": {
      "signature": "-----BEGIN SIGNATURE-----\nbase64...content...\n-----END SIGNATURE-----",
      "url": "https://github.com/org/repo/releases/download/v1.0.0/app-arm64-setup.nsis.zip"
    },
    "darwin-x86_64": {
      "signature": "-----BEGIN SIGNATURE-----\nbase64...content...\n-----END SIGNATURE-----",
      "url": "https://github.com/org/repo/releases/download/v1.0.0/app-x64.app.tar.gz"
    },
    "darwin-aarch64": {
      "signature": "-----BEGIN SIGNATURE-----\nbase64...content...\n-----END SIGNATURE-----",
      "url": "https://github.com/org/repo/releases/download/v1.0.0/app-aarch64.app.tar.gz"
    },
    "linux-x86_64": {
      "signature": "-----BEGIN SIGNATURE-----\nbase64...content...\n-----END SIGNATURE-----",
      "url": "https://github.com/org/repo/releases/download/v1.0.0/app-x86_64.AppImage.tar.gz"
    }
  }
}
```

### Required Fields
- `version` - semver (with or without 'v' prefix)
- `platforms.<target>.url` - download URL for binary
- `platforms.<target>.signature` - content of .sig file (base64)

### Optional Fields
- `notes` - release notes (markdown supported)
- `pub_date` - ISO 8601 timestamp

### Platform Keys
Standard Tauri platform identifiers:
- `windows-x86_64`, `windows-aarch64`
- `darwin-x86_64`, `darwin-aarch64` (macOS)
- `linux-x86_64`, `linux-aarch64`
- Optional: `{os}-{arch}-{installer}` (e.g., `windows-x86_64-nsis`, `windows-x86_64-msi`)

## 5. Signing: tauri signer generate

### Generate Key Pair
```bash
# Create keys
pnpm tauri signer generate -w ~/.tauri/myapp.key

# Output: creates two files
# ~/.tauri/myapp.key (PRIVATE - keep secret)
# ~/.tauri/myapp.key.pub (PUBLIC - distribute)
```

### Generated Files

**Private key format** (stored in `.key` file):
```
-----BEGIN PRIVATE KEY-----
base64...content...
-----END PRIVATE KEY-----
```

**Public key format** (stored in `.key.pub` file):
```
-----BEGIN PUBLIC KEY-----
base64...content...
-----END PUBLIC KEY-----
```

### Using Keys in tauri.conf.json
```json
{
  "plugins": {
    "updater": {
      "pubkey": "-----BEGIN PUBLIC KEY-----\nbase64...\n-----END PUBLIC KEY-----"
    }
  }
}
```

### Environment Variables for Signing

**During build/release:**
```bash
export TAURI_SIGNING_PRIVATE_KEY="$(cat ~/.tauri/myapp.key)"
export TAURI_SIGNING_PRIVATE_KEY_PASSWORD="optional-password-if-key-is-protected"
```

**In GitHub Actions:**
```yaml
env:
  TAURI_SIGNING_PRIVATE_KEY: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY }}
  TAURI_SIGNING_PRIVATE_KEY_PASSWORD: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY_PASSWORD }}
```

### CLI Options
```bash
tauri signer generate \
  -w ~/.tauri/myapp.key    # write to file
  -p "password"             # protect with password
  -f                        # force overwrite
  --ci                      # skip prompts
```

### Signature Flow
1. Build process reads private key from environment
2. Signs each platform binary → creates `.sig` file
3. Publishes binary + `.sig` file to release
4. `latest.json` includes signature content (base64)
5. Client verifies signature with public key before installing

## GitHub Actions Workflow Example

```yaml
name: Release

on:
  push:
    tags: [v*]

jobs:
  build:
    runs-on: ${{ matrix.os }}
    strategy:
      matrix:
        os: [ubuntu-latest, windows-latest, macos-latest]

    steps:
      - uses: actions/checkout@v3

      - uses: pnpm/action-setup@v2
      - uses: actions/setup-node@v3
        with:
          node-version: 22
          cache: pnpm

      - name: install dependencies
        run: pnpm install

      - name: build
        uses: tauri-apps/tauri-action@v1
        with:
          tagName: ${{ github.ref_name }}
          releaseName: Release ${{ github.ref_name }}
          releaseBody: See CHANGELOG
          releaseDraft: false
          prerelease: false
          uploadUpdaterJson: true
          uploadUpdaterSignatures: true
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          TAURI_SIGNING_PRIVATE_KEY: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY }}
          TAURI_SIGNING_PRIVATE_KEY_PASSWORD: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY_PASSWORD }}
```

## Quick Reference: API Call Flow

```typescript
// 1. Check for updates
const update = await check()

// 2. Handle result
if (update?.shouldUpdate) {
  // 3. Download with progress
  await update.download((progress) => {
    console.log(`Downloaded: ${progress.chunksDownloaded}/${progress.contentLength}`)
  })

  // 4. Install (triggers auto-restart on Windows)
  await update.install()
}

// 5. Always cleanup
await update?.close()
```
