# Brainstorm: Check for Updates - Localman Desktop App

**Date:** 2026-03-15
**Status:** Agreed
**Decisions:** GitHub Releases native + Background+Manual check + Toast+Dialog UX

---

## Problem Statement

Localman desktop app (Tauri v2) needs auto-update capability. Currently:
- `tauri-plugin-updater` v2.10.0 installed & initialized in Rust, but not wired up
- `tauri.conf.json` has empty `pubkey` and `endpoints`
- About section has stub "Check for Updates" button
- No JS binding (`@tauri-apps/plugin-updater`) installed
- Version hardcoded in 3 places (tauri.conf, package.json, about-section.tsx)

## Agreed Solution

### Delivery: GitHub Releases (native Tauri integration)
- Project migrating from GitLab (gitlabs.inet.vn) → GitHub public repo
- Tauri has built-in GitHub Releases support — zero custom endpoint code
- CI auto-generates `latest.json` manifest per platform
- Binary download direct from GitHub release assets (no auth needed)

### Check Timing: Background + Manual
- **Background:** Check on app startup + periodic (every 4h)
- **Manual:** "Check for Updates" button in Settings > About
- Respect offline-first: skip silently when offline

### UX Flow: Toast → Dialog
1. Update found → show toast notification ("Update v0.2.0 available")
2. User clicks toast → open Dialog with:
   - Version number + changelog/release notes
   - "Download & Install" button
   - "Later" dismiss button
3. Download progress shown (in-dialog or status bar)
4. Download complete → prompt restart
5. No update → toast "You're up to date" (manual check only)

### Platform Support: Windows + macOS + Linux
- CI build matrix for all 3 platforms
- `latest.json` includes platform-specific entries
- Platform keys: `windows-x86_64`, `darwin-x86_64`, `darwin-aarch64`, `linux-x86_64`

## Architecture

```
RELEASE PIPELINE (GitHub Actions on tag v*)
─────────────────────────────────────────
1. Generate signing keys (one-time)
   └─ tauri signer generate → private key (CI secret) + public key (tauri.conf)

2. CI Build Matrix
   ├─ Windows: .msi.zip + .msi.zip.sig
   ├─ macOS: .dmg + .app.tar.gz + .app.tar.gz.sig
   └─ Linux: .AppImage + .AppImage.sig

3. Tauri CLI auto-generates latest.json per platform
4. Create GitHub Release → upload all assets + latest.json

CHECK FLOW (App Runtime)
─────────────────────────
App start / 4h interval / manual button
  → GET https://github.com/{owner}/localman/releases/latest/download/latest.json
  → Tauri plugin compares version
  → If newer: Toast → Dialog → Download → Verify signature → Restart prompt
  → If same/older: No action (or "up to date" toast for manual)
```

## Implementation Plan

### Phase 1: Signing Setup (one-time)
- [ ] Run `tauri signer generate -w ~/.tauri/localman.key`
- [ ] Save private key as GitHub Actions secret: `TAURI_SIGNING_PRIVATE_KEY`
- [ ] Save password as secret: `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`
- [ ] Add public key to `tauri.conf.json` → `plugins.updater.pubkey`

### Phase 2: Tauri Config
- [ ] Set `plugins.updater.endpoints` to `["https://github.com/{owner}/localman/releases/latest/download/latest.json"]`
- [ ] Ensure `tauri.conf.json` version matches `package.json`

### Phase 3: Frontend Integration
- [ ] Install `@tauri-apps/plugin-updater` (pnpm add)
- [ ] Create `src/services/update-checker.ts`:
  - `checkForUpdates()` — calls plugin, returns update info or null
  - `downloadAndInstall()` — downloads + installs + prompts restart
- [ ] Create `src/hooks/use-update-checker.ts`:
  - Background check on mount + interval
  - State: checking, updateAvailable, downloading, progress
- [ ] Update `src/components/settings/about-section.tsx`:
  - Wire real check button to `checkForUpdates()`
  - Show version from Tauri API (remove hardcode)
- [ ] Create Update Dialog component:
  - Version, changelog, download progress
  - "Download & Install" / "Later" buttons
- [ ] Add toast notification for update available

### Phase 4: GitHub Actions CI/CD
- [ ] Create `.github/workflows/release.yml`:
  - Trigger: push tag `v*`
  - Matrix: windows-latest, macos-latest, ubuntu-latest
  - Steps: checkout → setup-node → pnpm install → tauri build (with signing)
  - Create GitHub Release → upload artifacts
- [ ] latest.json auto-generated by `tauri-action` GitHub Action

### Phase 5: Version Management
- [ ] Script to sync version across tauri.conf.json ↔ package.json ↔ Cargo.toml
- [ ] Remove hardcoded version from about-section.tsx (use Tauri API)

## Key Dependencies
- `@tauri-apps/plugin-updater` — JS binding for updater plugin
- `tauri-plugin-updater` — already in Cargo.toml ✓
- `tauri-apps/tauri-action` — GitHub Action for building + releasing Tauri apps
- Ed25519 signing keypair

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| Signing key leak | Malicious updates | Store only in GitHub Secrets, never commit |
| GitHub rate limit on releases API | Update check fails | Graceful fallback, skip silently |
| Large binary download on slow network | Bad UX | Show progress bar, allow cancel |
| Platform-specific build failures in CI | Missing platform updates | Build matrix with per-platform error handling |
| Version mismatch across config files | Wrong update detection | Automated version sync script |

## Alternatives Considered

| Approach | Verdict | Why |
|----------|---------|-----|
| GitLab Releases + backend proxy | Rejected | Migrating to GitHub, unnecessary complexity |
| GitLab Releases + static manifest | Rejected | Same as above |
| Custom update server | Rejected | YAGNI — GitHub native is sufficient |
| Sparkle/Squirrel (non-Tauri) | Rejected | Tauri has built-in updater, no need for external |
